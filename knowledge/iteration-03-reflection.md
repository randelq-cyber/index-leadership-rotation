# Iteration 03 Reflection

## Verdict
Iteration 03 confirmed that the project now has a **small, specific candidate rule family** rather than a broad exploratory idea.

## Main conclusions
### 1. T1 is the primary trigger
Compared directly against the control **T0**, **T1** remains the strongest primary trigger.

Most convincing cases:
- **60-day / 3-day**
- **60-day / 4-day**
- **60-day / 5-day**
- **252-day / 4-day**
- **252-day / 5-day**

The cleanest center of gravity looks around:
- **60-day high with 4-day hold**
- **252-day high with 4-day hold**

### 2. T5 is a real secondary trigger
T5 survived the tighter rerun.

It appears strongest in:
- **252-day / 3-day**
- **252-day / 4-day**
- **252-day / 5-day**
- with a smaller surviving case at **60-day / 5-day**

Interpretation:
T5 looks more like a selective confirmation-style trigger than a universal entry rule.

### 3. T0 remains useful as a benchmark, not the preferred trigger
The setup itself still carries information, especially at **252-day** highs. That means T0 should remain part of evaluation, but not necessarily the preferred candidate rule entry.

### 4. DIA-leading wildcard remains non-core
The **S2_T4_DIA_leads** branch still survives at **2-day** only, but it remains an asymmetric side-case, not part of the main rule family.

## What Iteration 03 resolved
The project now has a clear core structure:
- **Setup:** strict QQQ-leading / DIA-lagging non-confirmation under SPY context
- **Primary trigger:** T1 relative-strength turn
- **Secondary trigger:** T5 gap compression
- **Most credible horizon:** 4 to 5 trading days, with 3 days still relevant in some cases

## Surviving candidate rule family
### Primary candidate
- **S1_T1_QQQ_leads**
- strongest around **60-day / 4-day** and **252-day / 4-day to 5-day**

### Secondary candidate
- **S1_T5_QQQ_leads**
- strongest in the **252-day** cluster, especially **4-day to 5-day**

### Keep for benchmarking only
- **S1_T0_QQQ_leads**

### Keep as exploratory side-case only
- **S2_T4_DIA_leads** at **2-day**

## Bottom line
Iteration 03 says the project should now move from branch-pruning into **candidate-rule validation**.

The main rule family worth validating is:
- when **SPY confirms**, **QQQ leads**, and **DIA lags** under the strict setup,
- enter on a **T1 relative-strength turn**,
- with **4 to 5-day** holding windows as the main focus,
- and use **T5** as a secondary comparison branch, not the primary rule.
